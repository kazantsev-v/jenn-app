# Jenn — архитектура и идеи

## Философия

Jenn — это ядро, которое берёт на себя цифровые интерфейсы. Пользователь не думает **куда** сохранить — он просто пишет в Jenn, а Jenn сам разбирается: определяет намерение, выбирает категорию, направляет в нужный сервис.

---

## Полная цепочка

```
Input (tg/desktop/vk) → Core AI → Skill → Category AI → Output → Destination
```

### Пример

```
"Аргентина выиграла чм"

1. Router AI → { skill: "save_entry", params: { text: "Аргентина выиграла чм" } }
2. save_entry handler → Category AI → "Спорт"
3. Output Notion → database_id, привязанный к категории "Спорт"
4. Страница создана в Notion
```

---

## Компоненты

### 1. Inputs

Любой источник, который умеет слать `POST /v1/message`:

- Telegram bot
- VK bot
- Desktop input (Alt+Space)
- Discord bot
- CLI
- И любой другой, который реализует API

### 2. Core AI (Router)

Первый вызов AI. Получает текст + список скиллов. Возвращает `{ skill, params }`.

Провайдеры с fallback-цепочкой:
1. **Groq** — быстрый, бесплатный (`llama-3.3-70b`)
2. **OpenRouter** — запасной (`gpt-4o-mini`)
3. **GigaChat** — последний рубеж

Если все провайдеры упали → сообщение идёт в fallback-output (настроенный пользователем).

### 3. Skills

#### save_entry (универсальный скилл)

Единственный базовый скилл для сохранения информации. Несколько скиллов под каждый тип действия не нужно — один `save_entry` покрывает всё.

```json
{
  "name": "save_entry",
  "description": "Сохранить запись. AI сам определит категорию.",
  "params": {
    "text": "строка",
    "source_url": "строка (опционально)",
    "source_type": "строка (текст/ссылка/файл)"
  }
}
```

Handler:
1. Принимает текст от Router AI
2. Вызывает Category AI (внутренний запрос, дешёвой моделью):
   - Есть категории пользователя → выбрать подходящую
   - Нет подходящей → создать новую категорию
3. Передаёт текст + категорию в Output

#### Другие возможные скиллы

| Скилл | Назначение |
|-------|-----------|
| `translate` | Перевести текст |
| `summarize` | Сделать краткое содержание |
| `ask` | Ответить на вопрос (не сохранять) |

### 4. Category AI

Второй вызов AI — внутри `save_entry`. Определяет категорию для текста.

```
Prompt: "Текст: '{text}'. Категории: {list}. Выбери категорию или предложи новую. Ответь JSON: {\"category\": \"...\"}"
```

- Если есть категории пользователя → выбрать из списка
- Если категорий нет → AI придумывает название сам
- Если AI не уверен → `{ "category": null }` → буфер

### 5. Destinations

Каждый Output может содержать несколько **Destinations** — привязанные места назначения с шаблонами.

```json
{
  "outputs": {
    "notion": {
      "type": "notion",
      "auth": { "api_key": "..." },
      "destinations": {
        "notes": {
          "database_id": "abc123",
          "categories": ["Идеи", "Мысли", "*"],
          "template": {
            "Название": "{{title}}",
            "Содержимое": "{{content}}",
            "Дата": "{{received_at}}"
          }
        },
        "tasks": {
          "database_id": "def456",
          "categories": ["Задачи", "Спорт"],
          "template": {
            "Задача": "{{title}}",
            "Статус": "To Do"
          }
        },
        "buffer": {
          "database_id": "buffer789",
          "categories": ["*"],
          "template": {
            "Текст": "{{text}}",
            "Категория": "{{category}}"
          }
        }
      }
    }
  }
}
```

#### "Буфер" — страница по умолчанию

Если для текста не нашлось подходящей категории, или категория не привязана ни к одному destination — запись попадает в **буфер**. Это страница/таблица, куда падает всё неклассифицированное. Пользователь потом просматривает и распределяет вручную.

### 6. Категории (конфиг пользователя)

```json
{
  "categories": [
    { "name": "Нейросети", "keywords": ["AI", "нейро", "gpt", "llm", "искусственный интеллект"] },
    { "name": "Спорт",     "keywords": ["выиграл", "чемпион", "футбол", "матч", "турнир"] },
    { "name": "Идеи",      "keywords": ["идея", "можно сделать", "app", "стартап"] },
    { "name": "Новости",   "keywords": [] }
  ]
}
```

- `keywords` — подсказки для AI при классификации
- Если список пустой — AI определяет только по контексту

## Схема принятия решений

```
Сообщение: "Аргентина выиграла чм"

Router AI:
  └→ save_entry { text: "Аргентина выиграла чм" }

save_entry:
  └→ Category AI: "Спорт"
       ├→ Конфиг: категория "Спорт" → destination: notion.news
       │    └→ Notion → страница создана в database_id = abc
       │
       └→ Если категория не определена:
            └→ buffer → страница создана в database_id = buffer789
```

---

## Автоклассификация

### Как это работает

1. AI анализирует текст сообщения
2. Сравнивает с категориями пользователя (смотрит keywords + прошлые примеры)
3. Если уверен — сохраняет в соответствующую категорию
4. Если не уверен — сохраняет в буфер

### Обучение

- Пользователь перемещает запись из буфера в другую категорию → Jenn запоминает
- Со временем AI всё реже ошибается

## 7. Jenn Markdown Editor (родной Output)

### Концепция

**Jenn Markdown Editor** — это отдельное open-source приложение для заметок (альтернатива Obsidian).  
Оно не является частью Jenn Core, а подключается как один из Output-ов — самый совместимый, потому что и ядро, и редактор разрабатываются вместе.

```
Jenn Core                    Outputs
                      ┌──────────────────────┐
POST /v1/message      │ 🏆 Jenn Markdown     │ ← родной, 100% совместимость
  → AI → skill ───┬──▶│   Editor             │    граф, категории как папки,
                   │   └──────────────────────┘    VeraCrypt
                   │   ┌──────────────────────┐
                   │   │ 📄 Notion             │
                   │   └──────────────────────┘
                   │   ┌──────────────────────┐
                   └──▶│ 📊 Google Sheets      │
                       └──────────────────────┘
```

### Особенности

| Фича | Описание |
|------|----------|
| **Граф связей** | AI сам проставляет связи между записями. Сохранил статью про AI → Jenn связал со всеми похожими заметками |
| **Markdown storage** | Все заметки — `.md` файлы. Никакой проприетарной базы |
| **VeraCrypt интеграция** | «Красная кнопка» — зашифровать весь vault одним кликом |
| **Offline-first** | Редактор работает без интернета. Синхронизация с Jenn Core опциональна |
| **Категории как папки** | AI-категории = реальные папки с `.md` файлами |
| **Open source** | Полностью открыт. Любой может форкнуть |
| **Self-hosted** | npm install → готово. Никакого облака |

### Иерархия Outputs

| Output | Совместимость | Особенность |
|--------|--------------|-------------|
| 🏆 **Jenn Markdown** | 100% | Граф, папки-категории, шифрование, все фичи API |
| Notion | через API | Только базовые поля |
| Google Sheets | через API | Только строки |
| Telegram | Bot API | Переслать текст |

### Как это работает

1. Пользователь пишет в Telegram/Desktop: «идея: сделать приложение для поиска попутчиков»
2. Jenn Core → AI → `save_entry` → категория «Идеи»
3. Output `jenn_markdown` → создаёт файл `Идеи/2026-07-08-идея-приложение.md`
4. В редакторе заметка появляется с автоматическими связями к похожим заметкам

### Преимущество для конкурса

> **«Не просто AI секретарь. А AI секретарь + свой open-source редактор заметок с графами и шифрованием. Всё работает локально. Всё под твоим контролем.»**
